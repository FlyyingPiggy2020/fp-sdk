# utilities 实用工具组件库

本目录提供了一系列通用工具组件，为嵌入式系统开发提供各种功能支持，包括通信协议、信号处理、命令行界面、数据管理等。

## 文件结构

```
utilities/
├── bus/                  # 总线通信组件
├── butter/               # 巴特沃斯滤波器
├── data_center/          # 数据中心组件
├── shell/                # 命令行交互组件
├── export.c/h            # API导出机制
├── fplog.c/h             # 日志功能
├── heap.c/h              # 动态内存管理
├── pid.c/h               # PID控制器
├── pingpong.c/h          # 乒乓缓冲区
├── soft_timer.c/h        # 软件定时器
├── storage.c/h           # 存储功能
└── readme.md             # 本说明文件
```

## 核心组件说明

### 1. bus - 总线通信组件

**功能**：提供各种总线通信协议和模式的实现，支持全双工、半双工和Modbus等多种通信方式

**主要内容**：
- `serial_bus.c/h`: 串行总线公共实现，为各种总线通信提供基础支持
- `full/`: 全双工总线实现，支持同时发送和接收数据
- `half/`: 半双工总线实现，支持分时发送和接收数据
- `example/`: Modbus 485通信示例

**使用场景**：工业自动化、传感器网络、设备间通信等需要可靠数据传输的场景

### 2. butter - 巴特沃斯滤波器

**功能**：提供高质量的数字滤波器实现，支持多种滤波类型和参数配置

**主要内容**：
- `butter.c/h`: 滤波器核心实现，支持低通、高通、带通和带阻滤波
- `mybutter.m`: Matlab脚本，用于生成滤波器系数

**使用场景**：传感器信号处理、音频信号滤波、控制系统反馈信号平滑等需要消除噪声或提取特定频率分量的场景

### 3. data_center - 数据中心组件

**功能**：提供数据处理、分发和管理机制，支持数据订阅和发布模式

**主要内容**：
- `account.c/h`: 账号管理实现，支持数据的订阅和发布
- `data_center.c/h`: 数据中心核心实现，管理账号和数据流向
- `demo/`: 数据中心使用示例

**使用场景**：复杂系统中的数据管理、多模块间的数据共享、事件驱动系统等需要高效数据分发的场景

### 4. shell - 命令行交互组件

**功能**：提供轻量级的交互式命令行界面，支持命令解析、执行和扩展

**主要内容**：
- `shell.c/executor.c/parser.c/prompt.c`: Shell核心实现
- `cmds/`: 内置命令实现
- `inc/`: 头文件目录

**使用场景**：系统调试、配置管理、用户交互等需要命令行界面的场景

### 5. export.c/h - API导出机制

**功能**：提供统一的API导出机制，确保API的一致性和可移植性

**主要内容**：
- API导出宏定义
- 符号可见性控制
- 版本管理支持

**使用场景**：跨平台开发、库函数开发、API版本控制等需要统一接口管理的场景

### 6. fplog.c/h - 日志功能

**功能**：提供灵活的日志记录功能，支持不同级别的日志输出和格式化

**主要内容**：
- 日志级别定义（DEBUG、INFO、WARN、ERROR等）
- 日志格式化输出
- 日志重定向支持

**使用场景**：系统调试、错误追踪、运行状态监控等需要记录系统事件的场景

### 7. heap.c/h - 动态内存管理

**功能**：提供灵活的动态内存管理实现，支持自定义内存分配算法

**主要内容**：
- 内存分配和释放函数
- 内存池管理
- 内存使用统计

**使用场景**：需要动态内存分配的应用、内存资源受限的系统、需要内存使用统计的场景

### 8. pid.c/h - PID控制器

**功能**：提供经典PID控制器实现，支持位置式和增量式算法

**主要内容**：
- PID参数配置
- 位置式PID算法
- 增量式PID算法
- 抗积分饱和处理

**使用场景**：运动控制、温度调节、流量控制等需要闭环控制的场景

### 9. pingpong.c/h - 乒乓缓冲区

**功能**：提供高效的乒乓缓冲区实现，用于数据传输和处理

**主要内容**：
- 双缓冲区管理
- 数据同步机制
- 无锁访问支持

**使用场景**：高速数据采集、实时信号处理、DMA传输等需要高效数据缓冲的场景

### 10. soft_timer.c/h - 软件定时器

**功能**：提供软件定时器实现，支持多个定时任务和不同的时间单位

**主要内容**：
- 定时任务管理
- 回调函数机制
- 精度控制

**使用场景**：定时执行任务、周期性操作、延时控制等需要软件定时的场景

### 11. storage.c/h - 存储功能

**功能**：提供数据持久化存储功能，支持不同的存储介质和格式

**主要内容**：
- 存储设备抽象
- 数据读写API
- 错误检测和恢复

**使用场景**：配置信息存储、日志记录、数据备份等需要持久化存储的场景

| pingpong | 数据缓冲 | 高速数据采集、实时处理 |

## 使用建议

1. 根据实际需求选择合适的组件
2. 优先使用组件提供的API，避免直接修改组件内部实现
3. 注意组件间的依赖关系
4. 在资源受限的系统上，合理配置组件参数以平衡功能和性能
5. 遵循组件的使用示例和最佳实践

## 扩展开发

每个组件都设计为可扩展的，用户可以：
- 基于现有组件开发新功能
- 替换组件的底层实现以适应特定硬件
- 扩展组件API以满足特殊需求

详细的扩展开发指南请参考各组件的单独文档。
