# 动态内存管理组件

本目录提供了一个基于FreeRTOS heap_4.c实现的动态内存管理组件，支持内存分配、释放和使用情况查询。

## 文件结构

```
heap/
├── heap.c    # 动态内存管理实现
├── heap.h    # 动态内存管理头文件
└── readme.md # 本说明文件
```

## 核心功能

- 动态内存分配和释放
- 内存碎片管理
- 内存使用情况查询
- 线程安全的内存操作

## 核心组件说明

### 1. heap.h

**功能**：动态内存管理的头文件，包含所有公共API声明

**主要内容**：
- 内存分配和释放函数声明
- 内存使用查询函数声明

### 2. heap.c

**功能**：动态内存管理的实现文件，基于FreeRTOS heap_4.c实现

**主要内容**：
- 内存块管理算法
- 内存分配和释放逻辑
- 内存使用统计功能

## 基本使用方法

### 1. 内存分配

```c
#include "heap.h"

// 分配128字节内存
void *ptr = __heap_malloc(128);
if (ptr != NULL) {
    // 内存分配成功
    // 使用内存...
}
```

### 2. 内存释放

```c
// 释放之前分配的内存
__heap_free(ptr);
ptr = NULL;
```

### 3. 查询内存使用情况

```c
// 获取当前可用内存大小
size_t free_size = heap_get_free_remaining_size();

// 获取历史最小可用内存大小（内存峰值使用情况）
size_t min_free_size = heap_get_minimum_free_remaining_size();
```

## 内存管理算法

该动态内存管理组件基于FreeRTOS heap_4.c实现，采用以下算法：

1. **最佳适配算法**：寻找大小最接近请求的空闲内存块
2. **内存块合并**：释放内存时自动合并相邻的空闲块，减少内存碎片
3. **内存块头部**：每个内存块包含头部信息，记录块大小和使用状态

## 性能特点

- **线程安全**：支持多线程环境下的内存操作
- **低碎片**：通过内存块合并减少碎片产生
- **高效分配**：采用最佳适配算法提高内存利用率
- **实时性**：分配和释放操作的时间复杂度为O(n)，但常数项较小

## 配置选项

可以通过修改`options.h`文件中的相关宏来配置内存管理组件的行为：

- `FP_MEMORY_MANAGER`：选择内存管理算法（默认使用本组件）
- `__malloc`：内存分配宏，默认映射到`__heap_malloc`
- `__free`：内存释放宏，默认映射到`__heap_free`

## 使用建议

1. 尽量避免频繁的内存分配和释放，以减少内存碎片
2. 对于长期使用的内存，建议一次性分配足够的大小
3. 定期查询内存使用情况，避免内存泄漏
4. 在资源受限的系统上，注意控制内存分配的大小

## 与系统内存管理的关系

该组件提供了一套独立的内存管理接口，但可以通过宏定义与系统默认的内存管理算法进行切换。在`options.h`文件中，可以通过定义`FP_MEMORY_MANAGER`宏来选择使用系统默认的内存管理算法还是本组件提供的算法。
